# Regles projet TP2 (MVP)

## Contraintes fixes
- Projet choisi: Projet 2 - Pipeline de qualification de leads.
- Base de donnees: PostgreSQL.
- Contexte leads: uniquement besoins d'une boite informatique (services numeriques/IT).
- Cible: MVP fonctionnel, stable, compris de bout en bout.
- Scope: pas de version avancee dans ce rendu initial.

## Regles de developpement (a respecter a chaque session)
- Objectif clair.
- Prompts significatifs notes.
- Problemes rencontres -> solutions trouvees.
- Ce que j'ai appris.

## Finalisation obligatoire
- Tests des cas limites.
- Documentation.
- Polish minimal (messages d'erreur clairs, UI lisible).

## Livrables finaux attendus
- Code fonctionnel.
- README (installation + utilisation).
- Journal de developpement.
- Documentation des choix.

## MVP cible
- Import leads (CSV ou texte brut).
- Extraction structuree: nom, entreprise, besoin, budget, timing, source.
- Scoring explicite et reproductible.
- Statuts: nouveau, contacte, qualifie, perdu.
- Generation d'un email de premier contact personnalise.
- Persistance PostgreSQL.
